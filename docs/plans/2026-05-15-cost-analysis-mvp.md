# 成本分析软件第一版需求规格与开发计划

**目标：** 做一个 Win7 可用、轻量化、本地运行的成本分析软件，用于从报价单中提取内容，经过人工确认后生成客户可见的成本分析表。

**核心定位：** 软件内部可以保留导入追溯、模板识别、AI 建议、修改记录等辅助信息；导出给客户的成本分析表必须干净，只包含客户需要看到的固定字段。

**技术方向：** C# WinForms + SQLite + 轻量 Excel 读取组件，AI API 作为可选辅助模块预留。

---

## 1. 运行环境

- 操作系统优先支持 Windows 7。
- 软件应本地运行，不依赖浏览器内核。
- 软件应尽量轻量，避免 Electron、WebView2、大型前端框架。
- 数据保存在本地 SQLite 数据库。
- Excel 读取优先使用轻量库，必要时兼容 Excel COM。
- AI 功能通过 API 调用，不做本地大模型计算。
- AI 功能默认关闭，由用户手动配置和调用。

## 6. 报价单导入流程

导入流程：

1. 用户选择 Excel 报价单。
2. 软件扫描所有 Sheet。
3. 软件判断 Sheet 类型。
4. 软件找到表头行。
5. 软件识别数量阶梯行。
6. 软件识别数据起始行。
7. 软件提取物料行。
8. 软件生成待确认数据。
9. 用户确认或修改。
10. 用户填写客户总用量。
11. 软件匹配采购单价。
12. 用户确认加入成本分析表。

普通报价单识别规则：

- 如果同一行出现 `序号`、`产品型号` 或 `料号及名称`、`成品尺寸`、`材质/工艺`、`数量`，判断为普通报价单。
- 表头行下方如果出现多个数字，则判断为数量阶梯行。
- 数量阶梯行下方为数据起始行。
- 数据行从序号为数字的行开始，遇到 `以下空白` 或空行段落时停止。

- 产品编码

- 料号及名称

物料描述可能来源：

- 规格描述
- 成品尺寸
- 材质/工艺
- 材质工艺

采购单价可能来源：

- 报价

- 展开尺寸
- 成品尺寸
- 总长 / 总宽

如果 `料号及名称` 或 `产品型号` 同时包含编码和名称，软件应拆分：

示例：

```text
6-01.56.xxxxxx-0.002 ED-80D试剂包装白盒
```

拆分为：

```text
物料编码：6-01.56.xxxxxx-0.002
物料名称：ED-80D试剂包装白盒
```

## 8. 阶梯价格匹配规则

如果报价单只有一个单价：

```text
采购单价 = 该单价
```

如果报价单有单点数量阶梯：

```text
100 / 500 / 1000
```

默认选择不超过客户总用量的最大数量档。

示例：

| 客户总用量 | 采用价格档 |
| ---: | ---: |
| 80 | 100 档，并提示低于最低数量 |
| 100 | 100 档 |
| 300 | 100 档 |
| 500 | 500 档 |
| 800 | 500 档 |
| 1000 | 1000 档 |

如果报价单有区间数量阶梯：

```text
500-2499 / 2500-4999 / 5000-9999
```

按客户总用量命中对应区间。

## 9. 成本分析生成规则

每个物料生成一行客户成本分析明细。

基础规则：

```text
总价 = 采购单价 * 总用量
```

成本合计校验：

```text
成本合计 = 材料费 + 印刷费 + 后工序费 + 其他
```

如果 `成本合计` 与 `采购单价` 不一致，软件标记为计算异常，但允许用户确认保留。

如果报价单有成本分析 Sheet：

- 原材料成本优先映射为材料费。
- 加工成本优先映射为后工序费，具体拆分后续版本优化。
- 不良、营销、运费成本优先映射为其他。
- 报价优先映射为采购单价。

如果报价单只有最终报价：

- 采购单价取报价单价。
- 材料费、印刷费、后工序费、其他等待人工补充或后续由材料库、工艺库、AI 建议补全。

## 11. AI 辅助识别预留

AI 是第二识别方案，不替代本地规则。

主力 AI 供应商暂定为 DeepSeek。

DeepSeek 默认配置：

```text
API 地址：https://api.deepseek.com
默认模型：deepseek-v4-flash
高精度模型：deepseek-v4-pro
```

说明：

- DeepSeek API 使用与 OpenAI 风格相近的 Chat Completions 接口。
- JSON 输出场景应使用 `response_format: {"type":"json_object"}`。
- 系统提示词和用户提示词中必须明确要求只输出 JSON。
- 报价单识别第一阶段默认使用 `deepseek-v4-flash`，实际测试中响应更快；`deepseek-v4-pro` 保留为可选高精度模型。
- 请求应设置足够的 `max_tokens`，避免 JSON 被截断；第一版建议 `max_tokens=2000`。
- 旧模型名 `deepseek-chat`、`deepseek-reasoner` 不作为新开发默认值。
- API Key 存储时应做本地加密，不直接明文保存。
- Win7 直连 API 如遇 TLS 或网络兼容问题，后续可增加中转服务，但第一版不强制。

调用时机：

- 本地识别失败。
- 本地识别置信度低。
- 用户手动点击 AI 辅助识别。
- 成本字段缺失，需要建议补全。

AI 只返回结构化建议，不直接写入最终客户表。

AI 输出必须是 JSON。

AI 不允许决定：

- 最终采购单价
- 最终总用量
- 最终总价
- 是否导出客户表

所有金额和总价必须由本地程序重新计算。

21. 增加导入确认页原始文件预览：在导入确认页上方固定展示原报价单 Sheet 内容，下方展示识别结果，中间分隔条可拖动；原始预览高亮表头行、数量行和识别到的物料行，方便人工核对。

23. 增加阶梯价格结构化编辑：导入确认页可对选中物料打开“编辑阶梯价格”窗口，维护阶梯标签、最小数量、最大数量、单价；单价必须为数字，最大数量不能小于最小数量，保存后写回 `PriceTier` 列表并继续参与总用量匹配。
24. 增加主成本分析表阶梯价编辑：主表工具栏和右键菜单提供“编辑阶梯价”，可对当前明细行新增或修改阶梯价格，保存后立即按总用量重新匹配采购单价和总价。
25. 持久化阶梯价格：`cost_analysis_items` 增加 `price_tiers_json` 字段并自动迁移旧库；保存成本分析单时写入每行阶梯价格 JSON，打开历史单据时恢复到行数据并继续用于采购单价匹配。

27. 启动暂缓事项“完整工艺库自动计价”的第一阶段：新增本地工艺费用规则库，可按关键词匹配物料描述并自动填充材料费、印刷费、后工序费或其他费用；主表支持手动应用规则，导入报价单后也会自动尝试填充空白费用，且不覆盖已有人工金额。
28. 启动暂缓事项“历史成本参考”：新增本地历史成本查询，可按当前行物料编码精确匹配或物料名称模糊匹配已保存的成本分析明细；主表支持打开历史参考窗口，查看历史单号、客户、项目、成本拆分、采购单价、历史用量和阶梯价，并可将选中历史成本套用到当前行，套用时不带入历史总用量。
29. 启动暂缓事项“完整 AI 自动补全成本金额”的谨慎版：新增主表“AI补全成本”入口，按勾选行或存在空白费用的行发送给 DeepSeek；提示词包含当前明细、本地历史参考和工艺规则匹配摘要，要求返回严格 JSON。回填时只填空白的材料费、印刷费、后工序费、其他和采购单价，不覆盖已有人工金额，并用浅蓝高亮和 tooltip 标记 AI 建议来源。
30. 启动暂缓事项“复杂纸箱核价公式”：报价单导入器新增纸箱/外箱核价报价模板识别，支持“料号/名称/总长/总宽/基价/成品单价/计算公式”类表头，提取外箱料号、名称、展开尺寸、基价描述和成品单价；当成品单价为空但存在总长、总宽、基价时，可按 `(总长/25.4+加放英寸)*(总宽/25.4+加放英寸)*基价/1000` 计算单价建议，并识别表头中的 `+1`、`+2` 英寸加放。
31. 启动暂缓事项“PDF 自动解析 / 图片报价单 OCR”的轻量入口：导入文件选择器支持 PDF 和常见图片；PDF 会尝试在本地提取可复制文本并放入导入确认页原始预览，图片先进入待 OCR 预览入口。AI 辅助识别请求会附带原始预览文本，便于后续接视觉/OCR API 或人工复制文本识别。
32. 完善 PDF/图片到 AI 识别闭环：导入确认页新增“粘贴文本”，可把 PDF 复制文本、图片 OCR 文本或手工整理的报价内容放入顶部原始预览；当初始识别结果为 0 条物料时，AI 辅助识别返回的 `items` 会自动新增到识别结果表，并支持撤销回 AI 前状态。
33. 修复多材料识别：材质/工艺字段不再只取第一个片段作为材料名称，改为扫描整段文本并提取多个材料片段（如面纸、衬纸、双灰、灰板、双铜等），材料名称用 `；` 汇总；原材料克重也改为多值去重汇总（如 `157g；1250g`）。
34. 修复纸箱表识别失败时预览为空的问题：Excel 报价单即使未识别出明细，也会保留最相关 Sheet 的原始预览；纸箱核价模板会优先展示包含“料号/名称/总长/总宽/基价/单价”的计价公式页，而不是第一页基价明细。纸箱表头的“单价（元）/单价”也纳入纸箱单价列识别。
35. 增加纸箱专用“纸板基价表”模板：当 Sheet 内同时存在“品名/材质类型/基价”等字段时，导入确认页下方会按纸板材料行展示识别结果，而不是空白；基价有值则作为“基价(元/千平方英寸)”阶梯价带入，若该模板区域没有基价值则仍显示材料行供人工补价或 AI 识别。
36. 增加材料库和工艺规则的“扫描填入”：材料库可从当前成本分析表的材料名称、克重、材料厂家中扫描多材料候选并去重追加；工艺规则可从物料描述中扫描 `4C/UV/哑胶/啤/V槽/礼盒成型/磁铁/装箱/运输` 等关键词并带入费用类型。主表材料库匹配也支持多材料逐个匹配，多个材料只自动汇总厂家/规格，材料单价冲突时留给人工确认。
37. 增加材料库和工艺规则的独立批量扫描：两个库都可直接选择报价单文件夹批量读取 `.xls/.xlsx`，不需要先导入报价单到主成本分析表；扫描结果只追加到当前库表格并高亮，仍由用户确认后点击“保存”入库。
38. 优化材料库和工艺规则批量维护：独立批量扫描改为后台执行，显示进度条、当前文件和“停止扫描”；停止会在当前文件读取完成后中断后续文件，避免强杀 Excel COM。两个库都新增“批量编辑”，选中行时编辑选中行，未选中时编辑全部已有行，并支持“只填空白单元格”。
39. 启动供应商比价与“涨价红灯”报警系统第一阶段：导入确认页新增“价格预警”列，基于历史成本分析数据库比对同物料历史采购单价；同供应商价格上涨或其他供应商存在更低价时，使用黄色/红色单元格提示涨幅、历史价和更低供应商。
40. 补强 PDF 自动解析与图片 OCR：PDF 导入优先调用本机 `pdftotext` 提取版式文本，失败后回退内置 PDF 文本提取；图片导入优先调用本机 `tesseract` OCR；两者都支持同名 `.txt` 旁路文本。提取到文本后会先用本地规则解析物料编码、名称、尺寸、材质、克重和阶梯价，能直接在导入确认页生成明细，再交给 AI 或人工复核。PDF 内置文本会过滤扫描仪元数据、字体编码乱码和不可读片段，避免把乱码误显示为识别结果。
41. 继续补强复杂纸箱核价公式：纸箱核价模板支持“展开总长/总宽(英寸)+加放英寸”的英寸单位识别，不再把英寸重复除以 25.4；当行内没有基价但同 Sheet 前部存在纸板基价表时，会按材质类型回查基价并自动计算成品单价。导入结果的材质/工艺说明会追加基价、展开尺寸、公式加放和公式核价，方便人工复核。
42. 启动“如果…那么”式最低起订量模拟器：主表新增 `MOQ模拟` 入口，基于当前所有带阶梯价和总用量的明细，枚举阶梯价临界数量并计算整单当前成本、模拟成本、节省/增加金额、增加数量、影响明细和降价明细；结果窗口用绿色标记真正省钱的建议，用黄色标记单价下降但整单成本增加的情况，帮助谈判前判断是否值得提高起订量。
43. 增强供应商比价与涨价预警：价格预警不再只依赖物料编码或完整物料名称，还会读取最近历史明细，按物料关键词、成品尺寸、材料名称和克重做同类型规格匹配；当同供应商上浮、其他供应商更低或同类型规格历史价更低时，导入确认页继续用黄/红色提示。导入确认页新增“预警汇总”，可一次查看红色/黄色预警数量和每条明细的预警原因。
44. 启动 AI 未知单据批量自动清洗流水线第一阶段：扫描窗口新增 `AI清洗未知`，可对扫描后未识别/0 物料的报价单批量调用 DeepSeek；清洗成功后把 AI 结构化物料写回扫描结果。该能力后续已调整为进入 `识别校对` 窗口复核与学习，不再通过批量导入写入成本分析表。
45. 完成 AI 未知单据批量清洗后续阶段的本地模板沉淀：新增 `QuoteTemplateRepository` 使用已有 `quote_templates` 表保存学习模板；普通扫描成功和 AI 清洗成功后都会按表头关键词、模板类型、表头行、数据起始行等信息沉淀本地模板参考。后续 DeepSeek 报价单识别请求会自动附带相似模板提示，让同类未知单据更容易被稳定识别；规则仍作为辅助参考，最终以当前原始预览和人工确认结果为准。
46. 重新拉起 PDF 自动解析：外部提取服务升级为旁路 `.txt`、`pdftotext -layout`、`pdftotext -raw`、`pdftoppm+tesseract OCR` 的多级链路；外部提取结果会先做可读性过滤，明显乱码、扫描仪元数据和不可读符号不会再进入预览。文本解析器改为优先支持正常中文/英文报价字段，可识别物料编码、尺寸、材质/工艺、克重和 `500-2499=5.8` 这类阶梯价；对于图片型 PDF，如果本机没有 Poppler/tesseract，则明确提示安装工具或使用同名 `.txt` 旁路文本。
47. 补上 Win7 友好的 PDF/OCR 外部工具配置：新增 `OCR设置` 窗口和 `ocr_tool_settings` 本地表，可分别配置 Poppler 目录、`pdftotext.exe`、`pdftoppm.exe`、`tesseract.exe` 与 OCR 语言。外部提取服务查找顺序为：用户配置路径、程序目录下 `tools/poppler` / `tools/tesseract` 便携目录、系统 `PATH`，并提供检测状态，方便在不改系统环境变量的情况下启用图片型 PDF OCR。
48. 增加 Win7 运行环境检测：启动主窗体后自动检查操作系统版本、Windows 7 SP1、.NET Framework 4.8、系统/进程位数和内置 OCR 工具；侧边栏新增 `环境检测` 窗口，可重新检测并复制结果。若发现 Win7 未安装 SP1、.NET 4.8 缺失或 32 位系统导致 OCR 不可用，会用明确中文说明提示处理方式。
